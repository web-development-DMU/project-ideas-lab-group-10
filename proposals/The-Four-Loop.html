SourceFlow • Personal Shopping Request System

Personal Shopping
Request System

A web-based platform where customers submit sourcing requests, track progress, and receive concierge updates — making personalised shopping structured, traceable, and easy to manage.

Problem Personal shopping requests are often managed in DMs/spreadsheets with no consistent tracking.
Solution Centralised requests + statuses + notes, mapped cleanly to CRUD actions and routes.
Outcome Customers see progress; concierge updates faster; requests don’t get lost.

What we are building

SourceFlow is a request manager for personal shopping. Customers submit a request (item, size, budget, preferences), then track updates as the concierge team sources items, changes the status, and leaves notes.

Routes → Database →

Users and roles

  • Customer — create requests, edit details, view tracking/status.
  • Concierge/Admin — review requests, update statuses, add internal notes.

Later iterations can add authentication and permission-based access.

Core features

  • Create a shopping request with key details (item, size, budget, deadline).
  • Status tracking (e.g., New → In Progress → Sourced → Completed).
  • Concierge notes for internal updates and customer communication.
  • Simple admin management screens mapped to CRUD actions.

How it works (high level)

  • Customer submits a request.
  • Request is stored in the database with a default status.
  • Concierge updates status and adds notes during sourcing.
  • Customer views request progress at any time.
