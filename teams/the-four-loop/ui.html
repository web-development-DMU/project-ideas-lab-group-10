UI & Accessibility Documentation

This page documents the planned user interface for SourceFlow and explains how we will use semantic HTML and accessibility best practices to support all users.

UI goals

  • Clean, consistent navigation across pages.
  • Semantic HTML structure (headings, sections, forms, tables).
  • Accessible form design (labels, required fields, clear feedback).
  • Simple screens that map directly to CRUD actions.

User types

  • Customer: creates and edits their shopping requests.
  • Concierge: reviews requests, updates status, and adds notes.

We may begin with one interface and expose concierge actions as admin-only routes in later iterations.

Core pages and interactions

1) Home (GET /)

Landing page with calls-to-action: create a request and view requests.

2) New request form (GET /requests/new)

A form to create a new sourcing request.

  • Item name (required)
  • Brand (optional)
  • Budget (GBP) (optional)
  • Size (optional)
  • Colour (optional)
  • Notes (optional)

Submit sends a POST request to create the request.

3) Requests list (GET /requests)

Requests are displayed in a table view. Each request row includes key fields and actions.

  • Columns: Item, Brand, Budget, Status, Created, Actions
  • Actions: View/Edit, Update status, Delete

4) Request detail / edit (GET /requests/:id/edit)

Allows updating request details and (concierge-side) status and notes.

Semantic HTML plan

  • Use correct heading hierarchy: one <h1>, then <h2>/<h3>.
  • Use <section> and <article> to group content logically.
  • Forms: every input has a <label for> matching the input id.
  • Requests list uses <table> with <caption>, <thead>, and scoped headers.

Accessibility checklist

  • Keyboard navigation supported (logical tab order, visible focus).
  • Required fields indicated in text and using HTML validation (required).
  • Clear error feedback near inputs (no vague errors).
  • Status is not communicated by colour alone (text labels like “Pending”).
  • Images include meaningful alt text; decorative images use alt="".
