TheShopper System Architecture

This page describes the high-level architecture of TheShopper System, a personal shopping request platform developed by Emmanuel and Yazmin.

System Overview

  • Client: HTML, CSS, and JavaScript user interface
  • Server: JavaScript backend handling API routes
  • Database: SQLite database storing requests and users

Data Flow

  1. User submits a shopping request (item, budget, preferences).
  2. Server validates the request.
  3. Data is stored in the database.
  4. Admin (Emmanuel or Yazmin) reviews and updates the request.
  5. User can view the request status at any time.

Non-Functional Requirements

  • Usability: Simple and clear interface
  • Performance: Fast response using lightweight backend
  • Security: Admin access protected by password
  • Maintainability: Clean and modular code structure
